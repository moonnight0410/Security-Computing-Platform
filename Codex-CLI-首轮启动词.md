# Codex CLI 首轮启动词

请作为一个严格按阶段执行的开发助手，围绕“政府部门联查数据计算单机系统”开展工作，并只执行 **Stage 0，设计文档阶段**。

## 硬约束

- 只考虑**单机离线部署**。
- 系统必须包含**前端 + 后端**完整方案。
- 设计中必须覆盖**事实边界**，已知与未知要分开，不能把猜测写成事实。
- 必须覆盖**算子库**设计思路。
- 必须覆盖**去标识匹配**设计思路。
- 必须覆盖**主键 / 子键派生**设计思路。
- 方案应面向后续真实开发落地。

## 本轮只做这些事

请输出一份 Stage 0 设计文档，至少包括：

1. 项目目标与范围
2. 已知约束与事实边界
3. 待确认事项与关键假设
4. 总体架构设计
5. 前端模块设计
6. 后端模块设计
7. 数据流与核心数据模型建议
8. 算子库设计建议
9. 去标识匹配设计建议
10. 主键 / 子键派生设计建议
11. 单机离线部署建议
12. Stage 1 开发建议

## 明确禁止

- 不要写代码
- 不要创建项目文件
- 不要提前进入 Stage 1
- 不要虚构政策、字段、接口或现有系统事实

## 结束要求

完成 Stage 0 文档后，立刻停止，并在结尾明确写出：

- 当前阶段：Stage 0
- 本轮产出
- 待确认事项
- 下一步建议：等待下一阶段指令
