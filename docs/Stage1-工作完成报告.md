## 2. 阶段目标

Stage 1 的目标不是完成全部业务能力，而是建立后续可持续迭代的工程基础。

本阶段目标包括：

- 创建前端工程骨架。
- 创建后端工程骨架。
- 建立本地工作目录结构。
- 实现本域数据导入的基础入口。
- 实现字段画像的雏形。
- 实现不含数据的规则包登记。
- 实现规则包审批骨架。
- 实现本域任务草稿创建。
- 实现审计留痕入口。
- 将“本域数据不出域”作为系统级边界固化到界面和接口中。

## 3. 背景约束落实情况

### 3.1 已落实约束

| 约束 | Stage 1 落实方式 |
| --- | --- |
| 单机离线部署 | 前端和后端均运行在本机，后端监听 `127.0.0.1:8000`，前端监听 `127.0.0.1:5173` |
| 前端 + 后端 | 创建 `frontend/` 与 `backend/` 两个工程目录 |
| 本域数据不出域 | 数据文件只保存到本地 `workspace/imports/`，没有任何跨域传输接口 |
| 加密 / 哈希 / 去标识数据也不得出域 | Stage 1 不生成跨域哈希集合、密文集合、去标识键或布隆过滤器 |
| 规则可以流转 | 只允许登记“不含数据的规则包”元数据 |
| 输出受控 | 任务输出策略只保存为元数据，尚不生成真实对外文件 |
| 审计留痕 | 导入数据集、登记规则包、审批规则包、创建任务均写入审计记录 |

### 3.2 已确认治理选项

本阶段已按用户确认的 `BBB` 方案固化治理边界：

- 聚合统计：只允许单维粗粒度分组。
- 结论声明：执行人与审核人分离。
- 规则包：必须带签名引用，并经单名审批人批准后才能使用。

## 7. 本地工作流

```text
启动后端
  -> 启动前端
  -> 导入本域 CSV
  -> 查看字段画像
  -> 登记规则包

  -> 创建本域任务草稿
  -> 查看审计记录
```

## 8. 数据不出域控制

Stage 1 明确没有提供以下能力：

- 原始数据导出接口。
- 对象级明细查询接口。
- 哈希集合导出接口。
- 加密数据导出接口。
- 去标识键导出接口。
- 派生主键 / 子键导出接口。
- 跨域上传接口。
- 中心化汇聚接口。

Stage 1 允许保存或展示：

- 本域文件路径。
- 数据集行数和字段数。
- 字段画像摘要。
- 规则包元数据。
- 任务草稿元数据。
- 审计记录摘要。

## 10. 当前限制

- Stage 1 只完成基础骨架，不执行真实规则计算。
- CSV 可生成字段画像，Excel 只完成文件保存。
- 规则包只有元数据和审批状态，不解析规则表达式。
- 任务只是草稿，不执行主键派生、去标识处理或聚合统计。
- JSON 状态文件适合原型阶段，后续应切换 SQLite。
- 审计日志是普通 JSON 状态记录，尚未实现不可篡改机制。
