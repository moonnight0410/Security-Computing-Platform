### 2.2 规则表达式执行

规则包新增保存 `rules` 内容。

当前支持规则表达式：

```json
{
  "field": "benefit_status",
  "operator": "eq",
  "value": "正常"
}
```

支持操作符：

- `eq`
- `neq`
- `exists`
- `not_empty`
- `in`
- `gte`
- `lte`

执行结果只返回摘要：

- `rule_count`
- `rule_evaluated_count`
- `rule_pass_count`
- `rule_fail_count`
- `rule_unknown_count`

### 2.3 前端增强

规则包登记表单新增：

- 规则字段。
- 规则操作符。
- 规则值。
- 规则数量展示。

### 2.4 测试增强

后端测试从 9 个扩展到 12 个。

新增测试：

- 文本规则表达式通过 / 失败统计。
- 数值规则表达式通过 / 失败统计。
- 未知字段规则计入 unknown。

## 3. 数据不出域复核

Stage 4 没有放宽任何数据边界。

规则表达式执行仍然不返回：

- 通过的对象明细。
- 失败的对象明细。
- 字段原始值。
- 主键。
- 子键。
- 去标识摘要。

只返回数量摘要。

## 6. 当前限制

- SQLite 仍使用通用 records 表。
- 规则表达式不支持嵌套逻辑。
- 规则表达式不支持日期比较。
- 规则包签名引用尚未真实验签。
- 前端只支持录入单条规则。
