## 2. 项目简介

项目名称：`政府部门联查数据计算单机系统`

这是一个强调“本域数据不出域”的单机本地系统，核心目标不是跨部门交换原始数据，而是在受控本地环境中完成规则包流转、本地数据计算、执行留痕、安全输出和归档验签。

项目当前定位可以概括为：

- 规则包在本地导入、验签、审批、修订和治理
- 数据在本地导入和执行，不出域
- 结果只输出安全摘要，不输出对象级明细
- 全流程保留审计记录，可校验、可追溯

---

## 3. 建设背景与约束

项目面对的核心约束是：业务数据不能离开本域环境。

这意味着系统不能采用传统“多部门先交换数据、再集中联查”的方式，而必须改造成：

- 原始数据仅在本地导入和处理
- 原始数据、加密数据、哈希数据、去标识数据都不能直接出域
- 对外只能形成受控的执行回执、审核结论或满足阈值的聚合统计
- 所有关键动作都要保留审计留痕

因此，本项目解决的不是“数据汇聚”，而是“本地计算与安全治理”问题。

---

## 4. 当前系统形态

当前系统采用前后端分离的单机部署方式：

- 后端：`FastAPI`
- 前端：`React 19 + Vite + TypeScript`
- 本地状态存储：`SQLite`
- 本地审计链：`JSONL + 链式哈希`
- 输出与归档：本地文件落盘

核心目录如下：

```text
backend/    后端接口、服务逻辑、模型与测试
frontend/   前端页面、接口接入、样式与类型定义
docs/       设计方案、阶段报告、说明文档
scripts/    启动、停止、校验、签名辅助脚本
workspace/  本地运行态目录，包含导入、导出、归档、审计、日志与状态库
```

## 5. 系统已实现的核心能力

截至 `Stage 9`，系统已经形成一条较完整的本地治理主链。

### 5.1 数据导入与画像

- 支持本地导入 `CSV`、`XLSX`、`XLS`
- 支持 CSV 字段画像分析
- 支持统计字段名、空值、非空值、重复数、示例值等基础信息

### 5.2 字段映射与本地状态管理

- 支持主键字段映射
- 支持子键字段映射
- 支持敏感字段映射
- 支持聚合统计分组字段映射
- 本地状态已由 JSON 兼容层升级为 `SQLite`

### 5.3 本地任务执行

- 支持任务创建与执行
- 支持本地规则表达式计算
- 支持输出执行摘要
- 不返回对象级明细、不返回原始值、不返回派生键

### 5.4 输出边界控制

当前已支持 4 类输出策略：

- `local_only`
- `execution_receipt`
- `manual_assertion`
- `aggregate_summary`

其中聚合统计受以下边界约束：

- 仅允许单维粗粒度分组
- 最小阈值默认 `10`
- 小样本结果受抑制处理

### 5.5 规则包治理

- 支持规则包创建
- 支持规则包验签
- 支持规则包审批
- 支持批量验签与批量审批
- 支持规则表达式录入与执行统计

### 5.6 规则包中心与修订治理

这是当前阶段最重要的成果。

系统已将规则包从“登记对象”升级为“核心治理对象”，支持：

- 独立规则包中心
- 规则包草稿创建
- 规则包修订快照保存
- 已审批规则包自动派生新修订草稿
- 手动保存与离开页面自动保存
- 当前修订提交验签
- 审批通过后生效
- 废弃与受限删除
- 任务绑定具体规则包修订版本

### 5.7 输出审批与归档封存

- 支持输出申请
- 支持输出审批
- 支持输出包预览
- 支持输出文件本地落盘
- 支持多文件归档封存
- 支持归档清单、签名文件和验签报告

### 5.8 审计与追溯

- 关键动作写入本地审计记录
- 同步写入 `workspace/audit/audit-log.jsonl`
- 审计链包含 `previous_hash` 与 `entry_hash`
- 支持审计链完整性校验
- 支持本地应用日志落盘
