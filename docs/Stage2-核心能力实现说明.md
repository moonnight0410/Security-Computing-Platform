# Stage 2 核心能力实现说明

## 1. 当前阶段

当前阶段：Stage 2，核心能力实现阶段。

本阶段在“本域数据不出域”约束下实现最小可用核心链路：

```text
导入本域 CSV -> 保存字段映射 -> 登记并审批规则包 -> 创建任务 -> 本域执行 -> 查看摘要结果
```

## 2. 已实现能力

| 能力 | 当前实现 |
| --- | --- |
| 字段映射持久化 | 支持主键字段、子键字段、敏感字段和单维聚合字段映射 |
| 主键 / 子键派生 | 执行时在本域内存中派生，只输出完整数量和重复组数量 |
| 去标识处理 | 执行时在本域内存中生成摘要，只输出处理数量和重复组数量 |
| 算子库 | 提供标准化、主键派生、子键派生、去标识摘要、聚合阈值、执行回执等基础算子清单 |
| 任务执行 | 支持单数据集本域任务执行 |
| 聚合统计 | 支持单维粗粒度分组，并按最小阈值过滤 |
| 结论声明 | 支持生成“待审核”结论草稿，不直接输出对象级结论明细 |
| 执行回执 | 支持生成不含对象级数据的执行状态回执 |
| 结果查看 | 只展示摘要、回执、结论草稿、阈值后聚合统计和安全边界说明 |

## 3. 新增后端接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/operators` | 获取本域算子库清单 |
| GET | `/api/datasets/{dataset_id}/field-mapping` | 获取字段映射 |
| PUT | `/api/datasets/{dataset_id}/field-mapping` | 保存字段映射 |
| POST | `/api/tasks/{task_id}/execute` | 执行本域任务 |
| GET | `/api/results` | 获取执行结果摘要 |
| GET | `/api/tasks/{task_id}/result` | 获取指定任务结果摘要 |

## 4. 数据不出域控制

当前执行链路明确不返回：

- 原始字段值。
- 派生主键。
- 派生子键。
- 去标识摘要。
- 对象级命中明细。
- 阈值以下的聚合分组。

当前执行链路允许返回：

- 记录总数。
- 主键完整数量。
- 主键重复组数量。
- 子键完整数量。
- 子键重复组数量。
- 去标识处理数量。
- 去标识重复组数量。
- 执行回执。
- 待审核结论草稿。
- 满足阈值的单维聚合统计。

## 5. 当前限制

- 仅支持 CSV 执行，Excel 仍只支持文件落盘。
- 仅支持单数据集任务执行。
- 规则包当前仍是元数据与审批骨架，尚未解析具体规则表达式。
- 聚合统计只实现计数，不实现符合 / 不符合等业务口径。
- 结论声明只生成待审核草稿，不实现完整审核流。
- 状态存储仍使用 JSON 文件，后续建议切换 SQLite。
