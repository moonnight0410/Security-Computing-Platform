# Stage 1 基础骨架实现说明

## 1. 当前阶段

当前阶段：Stage 1，基础骨架实现阶段。当前项目已继续进入 Stage 2，本文件保留为 Stage 1 归档说明。

当前约束：所有数据只能在本域内处理，原始数据、加密数据、哈希数据、去标识数据、派生主键 / 子键和对象级明细结果均不得出域。

当前已确认治理选项为 `BBB`：

- 聚合统计只允许单维粗粒度分组。
- 结论声明必须执行人与审核人分离。
- 规则包必须带签名引用并经单名审批人批准。

本阶段最小闭环调整为：

```text
启动系统 -> 导入本域数据 -> 生成字段画像 -> 登记不含数据的规则包 -> 创建本域任务草稿 -> 记录边界审计
```

## 4. 当前限制

- CSV 支持字段画像，Excel 当前只完成文件落盘。
- 规则包当前只保存名称、用途、版本和规则数量，不执行规则。
- 任务只是草稿，不执行联查计算。
- 输出策略只是任务元数据，不会真正生成回执、结论或聚合统计结果。
- 结论声明只固化为“执行人与审核人分离”的边界，不实现完整审核工作流。
- 主键 / 子键派生、去标识处理、算子执行和规则引擎尚未实现。
- 本地状态使用 JSON 文件保存，后续可切换为 SQLite。

## 6. Stage 2 建议范围

进入 Stage 2 后建议优先实现：

1. 字段映射配置持久化。
2. 本域主键 / 子键派生的最小实现。
3. 本域去标识处理的最小实现。
4. 规则包到算子执行链路的最小实现。
5. 本域任务执行与本域结果查看。
6. 回执 / 结论 / 聚合统计输出能力的权限与边界控制。
7. 规则包签名校验和审批人记录。
