## 2. 本阶段完成项

### 2.1 规则包中心后端能力

- 新增规则包中心接口组。
- 新增规则包修订快照模型 `RulePackageRevision`。
- 新增规则包草稿保存载荷 `RulePackageDraftSave`。
- 新增规则包废弃载荷 `RulePackageDeprecate`。
- 新增规则包中心主对象与修订快照联动逻辑。
- 新增已审批规则包“编辑即自动派生修订草稿”能力。

### 2.2 规则包状态治理

- 草稿支持无签名保存。
- 每次保存都会生成新修订快照。
- 编辑后自动使旧签名失效。
- 当前修订提交验签后才可进入审批。
- 已审批规则包支持废弃。
- 仅草稿 / 失败规则包支持受限删除。

### 2.3 本域任务绑定修订

- 任务模型新增 `rule_package_revision_id`。
- 任务执行优先读取修订快照规则。
- 执行回执新增修订 ID。

### 2.4 前端规则包中心

- 新增独立 `RulePackageCenter` 组件。
- 首页新增“本域工作台 / 规则包中心”视图切换。
- 规则包中心支持：
  - 新建草稿
  - 编辑现有规则包
  - 多规则条目增删改
  - 手动保存
  - 页面离开自动保存
  - 提交验签
  - 审批通过
  - 废弃规则包
  - 删除草稿
  - 查看修订历史

### 2.5 测试与脚本

- 新增 `backend/tests/test_rule_package_revisions.py`。
- 后端测试总数由 34 增加到 39。
- 新增 `scripts/run-stage9-checks.ps1`。
- 前端版本号升级为 `0.9.0-stage9`。
- 后端 API 版本号升级为 `0.9.0-stage9-domain-local`。

## 3. 系统累计已实现功能总览

截至 Stage 9，系统已实现的全部功能如下。

### 3.1 本域数据导入与画像

- 支持本域导入 `CSV`、`XLSX`、`XLS` 文件。
- CSV 支持字段画像生成。
- 画像包含字段名、推断类型、空值数、非空数、重复数、样例值。
- 导入文件仅落盘于本域 `workspace/imports`。

### 3.2 字段映射与本地存储

- 支持主键字段映射。
- 支持子键字段映射。
- 支持敏感字段映射。
- 支持聚合分组字段映射。
- 本地状态由 SQLite 持久化。
- 旧 JSON 状态自动迁移并兼容补字段。

### 3.3 本域任务执行

- 支持本域任务草稿创建。
- 支持单数据集任务执行。
- 支持本域键材料补全统计。
- 支持重复组统计。
- 支持去标识摘要参与本域计算但不对外返回。
- 支持规则表达式计数型执行结果。

### 3.4 输出策略与结果边界

- 支持 `local_only`
- 支持 `execution_receipt`
- 支持 `manual_assertion`
- 支持 `aggregate_summary`
- 聚合统计限制为单维粗粒度分组。
- 聚合统计最小阈值控制为 `10`。
- 结果接口不返回对象级明细、原始键、派生键、去标识摘要。

### 3.5 规则表达式能力

- 支持平铺规则条目。
- 支持 `eq`
- 支持 `neq`
- 支持 `exists`
- 支持 `not_empty`
- 支持 `in`
- 支持 `gte`
- 支持 `lte`
- 仅输出通过 / 失败 / 未知数量，不返回命中明细。

### 3.6 规则包治理

- 支持规则包登记。
- 支持单个验签。
- 支持单个审批。
- 支持批量验签。
- 支持批量审批。
- 支持规则包中心独立管理。
- 支持规则包草稿创建。
- 支持规则包编辑。
- 支持规则包修订快照。
- 支持已审批规则包自动派生修订草稿。
- 支持规则包废弃。
- 支持规则包受限删除。

### 3.7 规则包签名与验签

- Stage 7 引入真实验签流。
- Stage 8 升级为 RSA 公私钥签名与公钥验签。
- Stage 9 保留 RSA 验签并与修订快照联动。
- 支持受信任签名人清单。
- 支持本地签名脚本。
- 支持编辑后签名失效。
- 支持草稿无签名保存。

### 3.8 结论声明与输出审批

- 支持结论声明待审核状态。
- 支持正式审核通过。
- 支持审核驳回。
- 支持执行人与审核人分离。
- 支持输出申请。
- 支持输出审批。
- 支持输出包预览。
- 支持输出文件本域落盘。

### 3.9 输出文件与归档封存

- 支持输出文件列表。
- 支持多文件归档封存。
- 支持生成归档清单。
- 支持归档签名文件。
- 支持归档验签报告。
- 支持归档报告重新验签。

### 3.10 审计与日志

- 所有关键动作写入本地审计表。
- 审计 JSONL 同步落盘。
- 审计链包含 `previous_hash` 和 `entry_hash`。
- 支持审计链完整性校验。
- 应用日志落盘到 `workspace/logs/app.log`。

### 3.11 前端工作台

- 支持本域工作台视图。
- 支持规则包中心视图。
- 支持数据导入、字段映射、任务、结果、审计、输出、归档操作。
- 支持规则包中心中的修订治理工作流。

### 3.12 脚本与验证

- 已提供后端、前端启动脚本。

## 5. 数据不出域复核

本阶段新增的规则包中心能力仍然严格遵守本域边界：

- 规则包只保存规则定义和治理元数据。
- 不包含任何对象级业务数据。
- 修订快照只保存规则文本与治理状态。
- 任务执行仍只返回摘要、回执、审批后的结论声明或满足阈值的聚合统计。
- 编辑、保存、验签、审批、废弃、删除均在本域内完成。

## 8. 阶段结论

当前阶段：Stage 9，规则包中心、修订快照与编辑治理阶段。

本轮产出不是在原有登记表单上打补丁，而是正式形成了规则包核心模块的第一版主链：

- 有独立规则包中心。
- 有主对象与修订快照。
- 有手动保存与离开自动保存。
- 有已审批规则包的派生修订逻辑。
- 有任务绑定修订。
- 有废弃与删除边界。

当前系统已经具备继续进入下一阶段的条件。下一阶段建议优先考虑：

1. 规则包修订差异对比。
2. 规则包引用关系查询与任务回溯页。
3. 规则模板复用与规则条目复用。
