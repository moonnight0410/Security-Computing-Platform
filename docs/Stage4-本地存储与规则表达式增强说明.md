# Stage 4 本地存储与规则表达式增强说明

## 1. 当前阶段

当前阶段：Stage 4，本地存储与规则表达式增强阶段。

本阶段在 Stage 3 完善与验证基础上继续增强两个落地能力：

- 使用 SQLite 替代 JSON 作为本地状态存储。
- 支持规则包中的最小规则表达式执行。

## 2. SQLite 本地存储

新增本地数据库：

```text
workspace/config/app-state.sqlite3
```

存储表：

```text
records(collection, record_key, payload, created_at, updated_at)
```

设计说明：

- 保留原有 `load_state`、`add_record`、`replace_collection` 接口，减少对业务代码影响。
- 首次发现旧 JSON 状态文件时自动迁移。
- 记录仍以 JSON payload 保存，便于当前原型快速迭代。
- 后续可逐步拆分为规范化关系表。

## 3. 规则表达式

规则包支持最小规则表达式：

```json
{
  "field": "benefit_status",
  "operator": "eq",
  "value": "正常"
}
```

当前支持操作符：

| 操作符 | 含义 |
| --- | --- |
| `eq` | 等于 |
| `neq` | 不等于 |
| `exists` | 字段存在且非空 |
| `not_empty` | 字段非空 |
| `in` | 属于列表 |
| `gte` | 数值大于等于 |
| `lte` | 数值小于等于 |

## 4. 规则执行输出

规则执行只输出摘要：

| 指标 | 含义 |
| --- | --- |
| `rule_count` | 规则数量 |
| `rule_evaluated_count` | 参与规则计算的行数 |
| `rule_pass_count` | 全部规则通过的行数 |
| `rule_fail_count` | 至少一条规则失败的行数 |
| `rule_unknown_count` | 规则字段缺失或无法判断的行数 |

不输出：

- 命中对象明细。
- 失败对象明细。
- 字段原始值。
- 主键、子键、去标识摘要。

## 5. 前端增强

规则包登记区新增：

- 规则字段。
- 规则操作符。
- 规则值。

规则包仍需：

- 签名引用。
- 单人审批通过。
- 审批后才可用于任务。

## 6. 测试增强

测试样例扩展为真实联查数据形态：

- 民政补贴资格核验。
- 人社就业状态核验。
- 医保参保状态核验。
- 重复主体。
- 缺失主键。
- 小样本分组。
- 规则表达式通过 / 失败 / 未知。

测试数量：`12` 个，已通过 `.\scripts\run-stage4-checks.ps1` 完整检查。

## 7. 当前限制

- SQLite 当前仍使用通用 records 表，尚未拆分为严格关系模型。
- 规则表达式只支持简单字段判断，不支持嵌套 AND / OR。
- 规则包签名只是引用字段，尚未进行真实签名验签。
- 前端当前只提供单条规则表达式录入。
