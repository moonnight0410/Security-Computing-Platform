# Stage 6 输出包落盘与审计链校验说明

## 1. 当前阶段

当前阶段：Stage 6，输出包受控落盘与审计链校验阶段。

本阶段承接 Stage 5 的安全输出审批能力，将“生成安全输出包预览”推进为“审批后写入本域受控文件”，并补齐审计链完整性校验能力。

## 2. 阶段边界

Stage 6 不改变数据不出域原则。

禁止写入输出文件的内容包括：

- 原始数据。
- 加密数据。
- 哈希数据。
- 去标识摘要。
- 派生主键 / 子键。
- 对象级明细。
- 低于阈值的聚合分组。
- 可反推个人或对象存在性的细粒度统计。

允许写入输出文件的内容仅限：

- 已审批执行回执。
- 已审批结论声明。
- 已审批且满足最小阈值的聚合统计。

## 3. 输出包落盘流程

```text
任务执行完成
  -> 创建安全输出申请
  -> 审批人与申请人分离审批
  -> 生成安全输出包
  -> 写入 workspace/exports
  -> 登记输出文件记录
  -> 写入链式审计记录
```

## 4. 输出文件记录

新增输出文件记录模型 `ExportFile`。

记录字段：

| 字段 | 说明 |
| --- | --- |
| `id` | 本域输出文件记录 ID |
| `request_id` | 对应输出申请 ID |
| `result_id` | 对应任务结果 ID |
| `export_type` | 输出类型：回执、结论声明、聚合统计 |
| `stored_path` | 本域文件路径 |
| `file_name` | 输出文件名 |
| `sha256` | 输出文件内容哈希，用于本域归档核验 |
| `byte_size` | 文件字节数 |
| `generated_at` | 文件生成时间 |
| `safety_notes` | 输出安全说明 |

## 5. 新增接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/export-files` | 查询本域输出文件记录 |
| POST | `/api/export-requests/{request_id}/file` | 将已审批安全输出包写入本域文件 |
| GET | `/api/audit/verify` | 校验审计 JSONL 链式哈希完整性 |

## 6. 审计链校验

审计链校验读取 `workspace/audit/audit-log.jsonl`。

校验内容：

- 每条记录必须是合法 JSON。
- 每条记录的 `previous_hash` 必须等于上一条记录的 `entry_hash`。
- 每条记录的 `entry_hash` 必须能由当前记录内容和 `previous_hash` 重新计算得到。

返回结果：

| 字段 | 说明 |
| --- | --- |
| `valid` | 审计链是否完整 |
| `total_entries` | 读取到的审计记录数 |
| `checked_entries` | 成功校验的审计记录数 |
| `first_invalid_index` | 首个异常行号 |
| `head_hash` | 当前链头哈希 |
| `errors` | 异常说明 |
