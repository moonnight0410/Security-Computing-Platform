# Stage 5 输出审批与审计防篡改说明

## 1. 当前阶段

当前阶段：Stage 5，输出审批与审计防篡改增强阶段。

本阶段目标是在允许输出执行回执、结论声明、聚合统计的前提下，补齐审批控制和安全输出包生成，避免用户绕过边界直接导出结果。

## 2. 安全输出流程

```text
任务执行完成
  -> 生成本域结果摘要
  -> 创建输出申请
  -> 审批人审批
  -> 生成安全输出包
```

## 3. 输出类型

| 类型 | 内容 | 边界 |
| --- | --- | --- |
| `receipt` | 执行回执 | 不包含对象级数据 |
| `assertion` | 结论声明 | 只输出待审核或已审核声明，不含明细 |
| `aggregate_summary` | 聚合统计 | 只输出满足阈值的聚合结果 |

## 4. 审批规则

- 输出申请必须记录申请人。
- 输出审批必须记录审批人。
- 审批人与申请人不能相同。
- 只有 `approved` 状态的输出申请可以生成输出包。

## 5. 审计链式哈希

`workspace/audit/audit-log.jsonl` 中新增：

- `previous_hash`
- `entry_hash`

每条审计记录基于上一条哈希和当前审计内容生成当前哈希，为后续防篡改校验提供基础。

## 6. 新增接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/export-requests` | 查询输出申请 |
| POST | `/api/export-requests` | 创建输出申请 |
| POST | `/api/export-requests/{request_id}/approve` | 审批输出申请 |
| GET | `/api/export-requests/{request_id}/package` | 生成安全输出包 |

## 7. 数据边界

输出包不包含：

- 原始数据。
- 派生主键。
- 派生子键。
- 去标识摘要。
- 对象级明细。
- 低于阈值的聚合分组。
