## 2. 本阶段完成内容

### 2.1 输出审批

新增模型：

- `ExportRequestCreate`
- `ExportRequestApprove`
- `ExportRequest`
- `ExportPackage`

新增能力：

- 创建安全输出申请。
- 审批安全输出申请。
- 校验审批人与申请人不能相同。
- 审批通过后生成安全输出包。

### 2.2 安全输出包

支持输出类型：

| 类型 | 输出内容 |
| --- | --- |
| `receipt` | 仅执行回执 |
| `assertion` | 仅结论声明 |
| `aggregate_summary` | 仅阈值后的聚合统计和抑制分组数量 |

输出包包含安全说明，明确不含原始数据、派生键、去标识摘要和对象级明细。

### 2.3 审计防篡改基础

审计 JSONL 追加：

- `previous_hash`
- `entry_hash`

该能力为后续审计链校验、归档验签、防篡改检测提供基础。

### 2.4 前端增强

新增“安全输出审批”区域：

- 选择输出类型。
- 填写申请人。
- 填写审批人。
- 填写用途。
- 创建输出申请。
- 审批输出申请。
- 预览安全输出包。

### 2.5 测试增强

新增测试覆盖：

- 未审批申请不能生成输出包。
- 审批人与申请人不能相同。
- 回执输出包只包含回执。
- 聚合输出包只包含阈值后的聚合统计。
- 无结论声明时不能生成结论输出包。
- 审计链条生成哈希字段。

## 4. 数据不出域复核

Stage 5 没有放宽任何数据边界。

新增输出包仍然禁止：

- 原始数据。
- 加密数据。
- 哈希数据。
- 去标识摘要。
- 派生主键 / 子键。
- 对象级明细。
- 低于阈值的聚合分组。
