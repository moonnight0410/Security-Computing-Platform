# Stage12 规则包导入与任务草稿完整化说明

## 设计原则

本阶段遵循两个原则：

1. 工作台只保留登记、执行、治理相关的核心动作，不在首页重复构造复杂编辑器。
2. 任务草稿必须成为真正可运作的核心模块，而不是只保留一个创建表单。

## 一、规则包登记为什么改为直接导入

原工作台中的规则包登记区允许手工填写：

- 规则字段
- 规则操作符
- 规则值

这种方式的问题是：

- 与规则包编辑页的职责重复。
- 无法体现“规则包作为资产导入”的治理方式。
- 对复杂组合规则完全不适用。
- 容易让首页重新变成编辑器堆叠页面。

因此本阶段改为：

- 工作台负责“导入登记”
- 规则包编辑页负责“创建、修订、组合、复用、审批治理”

## 二、工作台中的规则包导入流程

### 1. 用户动作

- 选择本地 JSON 规则包文件。
- 系统在前端读取文件并做预检。
- 预检成功后显示关键摘要。
- 用户点击“登记导入规则包”完成入库。

### 2. 预检内容

- `name`
- `version`
- `purpose`
- `signer_name`
- `signature_ref`
- `signature`
- `rules`
- `notes`

其中：

- `name` 和 `purpose` 为必需项。
- `rules` 必须为数组。
- `version` 缺省时自动补成 `0.1.0`。
- `signer_name`、`signature_ref` 缺省时优先使用系统内的默认签名人信息。

### 3. 示例结构

```json
{
  "name": "跨部门协查规则包",
  "version": "1.2.0",
  "purpose": "用于低保与救助协查任务的规则计算。",
  "signer_name": "市级规则中心",
  "signature_ref": "SIG-CENTER-001",
  "signature": "base64-signature",
  "notes": "仅用于本域任务联查。",
  "rules": [
    {
      "type": "group",
      "logic": "and",
      "children": [
        {
          "type": "rule",
          "field": "benefit_status",
          "operator": "eq",
          "value": "正常"
        },
        {
          "type": "rule",
          "field": "case_month",
          "operator": "gte",
          "value": "2025-01"
        }
      ]
    }
  ]
}
```

## 三、本域任务草稿为什么必须升级为核心模块

在当前系统中，任务草稿是从数据、规则到输出策略的承接层。若它只有“创建”功能，系统虽然能跑，但不具备真正可运作的工作流闭环。

因此本阶段将其升级为核心模块，补齐如下能力：

- 创建
- 载入编辑
- 保存修改
- 另存为新草稿
- 删除
- 直接执行
- 查看关联执行结果

## 四、本域任务草稿的完整工作流

### 1. 创建草稿

用户可配置：

- 任务名称
- 任务说明
- 数据集
- 规则包
- 输出策略
- 聚合维度
- 最小阈值

### 2. 编辑草稿

- 任务列表中的任意任务都可加载到表单中。
- 表单进入编辑态后，可直接保存覆盖当前任务。

### 3. 另存为新草稿

- 对已存在任务进行参数变体化时，不覆盖原任务。
- 直接复制当前表单配置生成新任务。

### 4. 删除草稿

- 对未产生执行结果的任务允许删除。
- 对已产出结果的任务，后端阻断删除与修改，避免破坏结果链路。

### 5. 执行草稿

- 可在草稿列表内直接执行。
- 执行后刷新：
  - 任务状态
  - 更新时间
  - 结果联动信息

### 6. 结果联动

草稿模块中新增了与结果的联动摘要，能直接看到：

- 当前选中任务
- 最近更新时间
- 是否已生成结果
- 当前结果状态

## 五、任务对象后端补全

### 新增字段

- `updated_at`

### 新增接口

```text
PUT    /api/tasks/{task_id}
DELETE /api/tasks/{task_id}
```

### 保护规则

- 若任务已经生成执行结果，则不允许直接修改或删除。
- 需要基于已有任务继续调整时，应使用前端“另存为新草稿”能力生成新任务。

```powershell
scripts/run-stage12-checks.ps1
```

- 后端 `compileall`

## 七、当前阶段价值

本阶段完成后，工作台中的两个关键对象已经清晰分层：

- 规则包：导入登记与治理
- 任务草稿：配置、迭代、执行与联动结果

这使系统更接近正式产品形态，而不是演示级表单集合。

## 八、下一个 Stage 建议方向

建议下一个 Stage 继续沿“治理闭环”和“正式产品化”两个方向推进：

### 1. 规则包导入治理增强

- 增加导入模板下载能力。
- 增加导入失败明细反馈。
- 增加导入记录与导入来源留痕。

### 2. 任务草稿生命周期增强

- 增加任务复制来源标识。
- 增加只读历史任务视图。
- 增加任务删除前影响提示。

### 3. 任务-结果-输出联动增强

- 在任务列表中直接看到最近一次执行摘要。
- 允许从任务跳转到输出申请和归档对象。
- 在治理看板中增加任务到输出的链路追踪。

### 4. 验收样例与测试增强

- 补充规则包导入异常样例。
- 补充任务更新/删除保护样例。
- 补充输出审批与归档的串联样例。
