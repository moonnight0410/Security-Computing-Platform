# Stage 0 设计文档：政府部门联查数据计算单机系统

## 1. 项目目标

构建一个用于政府部门联查数据计算的单机离线系统，支持前端 + 后端完整交付，在不依赖外网服务的前提下完成数据导入、规则计算、去标识匹配、主键 / 子键派生、结果输出与过程留痕。

本阶段仅输出设计文档，不进行代码实现、数据库建表、接口编写或页面开发。

### 1.1 核心目标

| 目标 | 说明 |
| --- | --- |
| 多部门数据导入与管理 | 支持将不同来源的结构化数据导入本地系统，进行字段识别、预览和管理 |
| 算子库计算 | 通过可配置算子完成匹配、校验、统计、异常识别等计算 |
| 去标识匹配 | 在尽量不暴露原始敏感标识的前提下完成对象关联 |
| 主键 / 子键派生 | 基于统一规则生成稳定、可追溯的数据关联键 |
| 全过程留痕 | 保留导入、配置、计算、导出等关键操作记录 |
| 单机离线运行 | 所有数据、配置、计算、结果均在本机完成 |

### 1.2 本阶段不做事项

- 不创建前端项目。
- 不创建后端项目。
- 不编写数据库脚本。
- 不编写接口代码。
- 不实现页面。
- 不内置未确认的业务政策规则。
- 不假设存在外部政务平台接口。

## 2. 已知约束

| 约束类型 | 已知约束 |
| --- | --- |
| 部署模式 | 仅考虑单机离线部署 |
| 系统形态 | 必须包含前端 + 后端，不是纯脚本工具 |
| 数据处理 | 必须覆盖算子库设计 |
| 数据关联 | 必须覆盖去标识匹配机制 |
| 键体系 | 必须覆盖主键 / 子键派生规则设计 |
| 安全表达 | 只做通用工程设计，不虚构政策条文 |
| 可扩展性 | 支持未来新增部门、新增算子、新增匹配规则 |
| 阶段边界 | 当前只执行 Stage 0 设计文档阶段 |

## 3. 事实边界与假设项

### 3.1 已知事实

- 当前项目目标是“政府部门联查数据计算”的单机系统。
- 系统需要支持前端和后端。
- 系统需要支持离线部署和运行。
- 系统需要覆盖算子库、去标识匹配、主键 / 子键派生。
- 系统输出应面向后续真实开发落地。

### 3.2 未知事实

| 未知项 | 影响 |
| --- | --- |
| 真实业务事项 | 影响规则模板、结果口径和页面流程 |
| 数据来源部门 | 影响字段映射、数据质量校验和权限边界 |
| 输入文件格式 | 影响导入模块和解析能力 |
| 字段清单 | 影响主键、子键、去标识键设计 |
| 敏感字段范围 | 影响遮蔽、导出、审计和加密策略 |
| 数据规模 | 影响本地数据库、任务调度和性能设计 |
| 使用人员角色 | 影响权限模型和操作流程 |
| 部署操作系统 | 影响打包方式、启动脚本和本地存储路径 |

### 3.3 关键假设

| 假设项 | 说明 | 后续确认方式 |
| --- | --- | --- |
| 输入以结构化文件为主 | 先按 Excel、CSV 等文件导入设计 | Stage 1 前确认样例文件 |
| 计算任务以批处理为主 | 单机离线环境更适合批量导入、批量计算、批量导出 | Stage 1 前确认处理流程 |
| 规则先配置化，不硬编码业务政策 | 避免虚构业务事实，便于扩展 | Stage 2 前确认规则样例 |
| 敏感字段默认最小化展示 | 降低误展示风险 | Stage 1 前确认敏感字段策略 |
| 本地用户权限先做轻量设计 | 单机系统优先保证可用性和审计 | Stage 3 前确认是否需要多用户 |

## 4. 总体架构

### 4.1 推荐架构

建议采用“本地 Web 前端 + 本地后端服务 + 本地数据库 + 本地文件存储 + 本地计算引擎”的结构。

```text
用户浏览器
  |
  | 本机访问
  v
前端应用
  |
  | 本地 API
  v
后端服务
  |
  +-- 文件导入与解析
  +-- 字段映射与数据画像
  +-- 主键 / 子键派生
  +-- 去标识处理
  +-- 算子库与规则引擎
  +-- 匹配计算任务
  +-- 结果导出
  +-- 审计日志
  |
  v
本地数据库 + 本地文件目录
```

### 4.2 为什么采用该架构

- 单机离线场景下，本地前后端分离架构便于后续扩展页面和 API。
- 后端集中处理数据导入、计算和审计，避免前端直接处理大量敏感数据。
- 本地数据库保存配置、任务、结果索引，本地文件目录保存原始文件和导出文件，更适合大文件处理。
- 算子库和规则引擎独立成计算层，便于后续新增规则和算子。

### 4.3 备选方案

| 方案 | 说明 | 是否推荐 |
| --- | --- | --- |
| 桌面客户端一体化 | 前端和后端打包成桌面应用 | 可作为后续打包方案 |
| 纯脚本工具 | 用命令行完成导入计算 | 不推荐，不满足前端 + 后端约束 |
| 单体 Web 应用 | 前后端在一个服务内渲染 | 可行，但页面交互扩展性较弱 |

## 5. 前端设计

### 5.1 前端模块划分

| 模块 | 功能 | Stage 1 建议 |
| --- | --- | --- |
| 工作台 | 展示任务、数据集、最近结果 | 先做基础入口 |
| 数据导入 | 选择本地文件、预览表头和样本数据 | Stage 1 必做 |
| 字段映射 | 配置字段角色、敏感级别、输出策略 | Stage 1 必做 |
| 主键 / 子键配置 | 配置单字段键、组合键、派生键规则 | Stage 1 建立框架 |
| 去标识配置 | 配置参与字段、哈希策略、遮蔽策略 | Stage 1 建立框架 |
| 算子库管理 | 展示可用算子、参数、说明 | Stage 1 建立空模块 |
| 任务配置 | 选择数据集、匹配策略、校验规则 | Stage 1 建立基础表单 |
| 结果查看 | 查看命中、未命中、异常、统计 | Stage 1 可做占位 |
| 审计导出 | 查看操作记录、导出结果和任务摘要 | Stage 1 可做占位 |

### 5.2 前端交互原则

- 数据导入后先预览，不直接进入计算。
- 字段映射必须由用户确认，系统只提供候选建议。
- 涉及敏感字段时，默认遮蔽展示。
- 匹配结果应明确区分“计算结果”和“业务结论”。
- 所有关键配置应支持保存和复用。

## 6. 后端设计

### 6.1 后端模块划分

| 模块 | 职责 | 设计理由 |
| --- | --- | --- |
| 文件解析模块 | 解析 Excel、CSV 等结构化文件 | 数据导入是所有计算前提 |
| 数据画像模块 | 统计字段类型、空值率、重复率、样本值 | 帮助用户判断字段质量 |
| 字段映射模块 | 保存字段角色、敏感等级、输出策略 | 避免硬编码字段含义 |
| 主键派生模块 | 生成主键、组合键、子键 | 保证跨表关联规则一致 |
| 去标识模块 | 标准化、加盐、哈希、遮蔽 | 降低敏感标识暴露 |
| 算子库模块 | 管理基础算子和组合算子 | 支撑规则计算可扩展 |
| 规则引擎模块 | 执行条件判断和校验规则 | 保证计算过程可解释 |
| 匹配引擎模块 | 执行精确匹配、组合匹配、去标识匹配 | 支撑联查核心能力 |
| 任务管理模块 | 创建、运行、终止、记录任务状态 | 适配批处理计算 |
| 结果管理模块 | 保存结果、统计摘要、异常信息 | 支撑查询和导出 |
| 审计模块 | 记录操作、配置、任务、导出行为 | 支撑追溯 |

### 6.2 后端设计原则

- 所有计算规则配置化。
- 所有任务执行可追溯。
- 所有导出结果可定位到任务、配置和输入数据。
- 算子和规则区分：算子是基础能力，规则是业务组合。
- 不在 Stage 0 设定具体数据库表，只给出模型建议。

## 7. 数据模型建议

### 7.1 核心模型

| 模型 | 关键字段建议 | 说明 |
| --- | --- | --- |
| Dataset | id、名称、来源文件、导入时间、行数、字段数、状态 | 表示一次导入的数据集 |
| FieldProfile | 数据集 id、字段名、类型推断、空值率、重复率、样本值 | 表示字段画像 |
| FieldMapping | 数据集 id、原始字段名、字段角色、敏感级别、标准化策略 | 表示字段映射配置 |
| KeyRule | 名称、键类型、参与字段、标准化方式、派生方式、版本 | 表示主键 / 子键规则 |
| DeIdRule | 名称、参与字段、盐值策略、哈希算法、输出策略、版本 | 表示去标识规则 |
| Operator | 名称、分类、参数定义、输出类型、版本、说明 | 表示算子库条目 |
| RuleConfig | 名称、算子组合、参数、触发条件、严重级别 | 表示规则配置 |
| MatchTask | 名称、数据集范围、匹配策略、规则配置、状态 | 表示计算任务 |
| MatchResult | 任务 id、匹配状态、匹配键、差异摘要、规则结果 | 表示计算结果 |
| AuditLog | 操作人、操作时间、操作类型、对象类型、摘要 | 表示审计日志 |

### 7.2 数据存储建议

- 原始导入文件保存在本地文件目录。
- 结构化解析结果可进入本地数据库或中间文件。
- 配置、任务、结果摘要、审计日志进入本地数据库。
- 大体量结果明细可保存为本地文件，并在数据库中保存索引。

## 8. 算子库设计建议

### 8.1 算子分类

| 分类 | 示例 | 用途 |
| --- | --- | --- |
| 标准化算子 | 去空格、大小写统一、日期格式统一 | 提高匹配稳定性 |
| 匹配算子 | 等值匹配、组合键匹配、去标识键匹配 | 完成跨表关联 |
| 比较算子 | 等于、不等于、大于、小于、区间判断 | 完成条件校验 |
| 集合算子 | 包含、不包含、交集、差集、重复检测 | 完成名单类、集合类判断 |
| 日期算子 | 有效期判断、起止日期比较、期限计算 | 处理时间条件 |
| 统计算子 | 计数、去重计数、分组汇总、比例计算 | 生成汇总结果 |
| 异常算子 | 空值异常、重复异常、格式异常、冲突异常 | 发现数据质量问题 |
| 文本辅助算子 | 名称相似度、地址相似度 | 仅作为辅助提示，不默认作为事实结论 |

### 8.2 算子元数据

每个算子建议包含：

- 算子名称。
- 算子编码。
- 算子分类。
- 输入参数定义。
- 输出结果定义。
- 是否允许处理敏感字段。
- 是否可用于去标识数据。
- 版本号。
- 说明与限制。

### 8.3 算子库设计理由

- 将基础计算能力沉淀为算子，避免后续业务规则重复开发。
- 算子版本化后，历史任务可追溯。
- 算子参数化后，非开发人员可通过配置组合规则。
- 将敏感字段处理能力纳入算子元数据，可降低误用风险。

## 9. 去标识匹配设计建议

### 9.1 基本流程

```text
原始字段
  -> 字段标准化
  -> 字段组合
  -> 加盐或密钥处理
  -> 哈希生成去标识键
  -> 使用去标识键匹配
  -> 输出最小化结果
  -> 记录审计摘要
```

### 9.2 去标识匹配策略

| 策略 | 说明 |
| --- | --- |
| 标准化优先 | 哈希前先清洗格式，避免同一对象因格式差异无法匹配 |
| 固定字段顺序 | 组合字段必须有固定顺序，保证结果稳定 |
| 盐值管理 | 盐值不应与结果文件一起明文导出 |
| 最小化输出 | 默认只输出命中状态、摘要和必要字段 |
| 版本化配置 | 去标识规则变化必须产生新版本 |

### 9.3 边界说明

- 去标识不等于匿名化。
- 去标识键仍可能在特定条件下被关联分析。
- 如果参与字段质量不足，可能出现漏匹配。
- 如果弱唯一字段组合不合理，可能出现误匹配。
- Stage 0 不设计跨机构在线隐私计算，只设计单机本地去标识匹配。

## 10. 主键 / 子键派生设计建议

### 10.1 主键类型

| 类型 | 说明 | 适用场景 |
| --- | --- | --- |
| 单字段主键 | 单个字段可稳定唯一识别对象 | 字段质量高且唯一性明确 |
| 组合主键 | 多个字段组合后识别对象 | 单字段不足以唯一识别 |
| 派生主键 | 标准化或哈希后生成内部键 | 需要减少原始标识暴露 |

### 10.2 子键类型

| 类型 | 示例表达 | 用途 |
| --- | --- | --- |
| 对象 + 时间 | 主体键 + 年月 | 区分同一对象不同周期记录 |
| 对象 + 事项 | 主体键 + 事项编码 | 区分同一对象不同业务事项 |
| 对象 + 证照 | 主体键 + 证照类标识 | 区分同一对象不同证照记录 |
| 对象 + 来源 | 主体键 + 数据来源 | 区分同一对象跨部门记录 |

以上示例仅为键设计模式，不代表真实字段已存在。

### 10.3 派生规则

- 先标准化，再拼接，再派生。
- 拼接顺序必须固定。
- 空值处理必须明确。
- 派生规则必须记录版本。
- 主键用于主体级关联，子键用于主体下细分记录关联。
- 弱唯一字段不能单独作为主键，只能作为辅助字段或组合字段。

## 11. 单机离线部署建议

### 11.1 部署组成

| 组成 | 建议 |
| --- | --- |
| 前端 | 本地浏览器访问的 Web 前端 |
| 后端 | 本机运行的 API 服务 |
| 数据库 | 本地轻量数据库或嵌入式数据库 |
| 文件目录 | 保存导入文件、结果文件、审计包 |
| 启停方式 | 提供一键启动和停止脚本 |
| 升级方式 | 使用离线升级包 |

### 11.2 本地目录建议

```text
workspace/
  imports/      原始导入文件
  parsed/       解析后中间数据
  results/      计算结果
  exports/      用户导出文件
  logs/         运行日志
  audit/        审计摘要
  config/       本地配置
```

此处为部署目录建议，不代表本阶段创建项目目录。

### 11.3 安全建议

- 默认仅允许本机访问。
- 默认不启用外网连接。
- 敏感字段默认遮蔽。
- 导出结果使用字段白名单。
- 密钥、盐值、任务数据分开保存。
- 重要操作写入审计日志。
- 删除操作应区分原始文件、结果文件和任务记录。

## 12. 风险与待确认项

### 12.1 主要风险

| 风险 | 说明 | 建议 |
| --- | --- | --- |
| 字段质量不足 | 缺失、格式混乱、重复会影响匹配准确性 | Stage 1 先实现字段画像 |
| 主键规则不明确 | 键规则不稳定会导致结果不可追溯 | 规则版本化 |
| 去标识误解 | 去标识可能被误认为完全匿名 | 文档和界面明确提示边界 |
| 规则被硬编码 | 后续业务变化会导致维护困难 | 算子和规则配置分离 |
| 数据规模未知 | 单机性能可能不足 | Stage 1 前确认数据量级 |
| 审计要求不清 | 影响日志粒度和归档方式 | Stage 3 前确认审计要求 |

### 12.2 待确认事项

- 是否已有样例数据文件。
- 输入文件主要格式是什么。
- 单次任务预计数据量级。
- 是否需要多用户登录和角色权限。
- 是否需要导出完整明细还是仅导出统计结果。
- 敏感字段有哪些类型。
- 是否需要内置初始算子清单。
- 是否需要兼容特定操作系统。
- 是否需要离线安装包。

## 13. Stage 1 开发范围建议

只有用户明确要求进入 Stage 1 后，才执行以下内容。

### 13.1 Stage 1 建议目标

建立可运行的前后端基础骨架，完成最小闭环：

```text
启动系统 -> 导入文件 -> 预览字段 -> 保存字段映射 -> 创建空任务 -> 查看任务列表
```

### 13.2 Stage 1 建议范围

| 范围 | 内容 |
| --- | --- |
| 项目骨架 | 创建前端、后端、文档、配置目录 |
| 前端基础 | 工作台、数据导入页、字段映射页、任务列表页 |
| 后端基础 | 健康检查、数据集接口、字段画像接口、任务接口 |
| 本地存储 | 配置本地数据库或文件型存储 |
| 文件导入 | 实现 CSV / Excel 的基础解析能力 |
| 审计雏形 | 记录导入、映射、任务创建操作 |
| 文档 | 补充本地启动说明 |

### 13.3 Stage 1 暂不建议做

- 完整算子库实现。
- 复杂规则引擎。
- 去标识匹配完整实现。
- 主键 / 子键复杂派生。
- 大规模性能优化。
- 打包安装程序。
- 真实业务规则模板。
